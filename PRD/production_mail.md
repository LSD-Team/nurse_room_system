# Production Email Deployment Guide

เอกสารนี้บันทึกขั้นตอนการเปลี่ยนระบบส่งอีเมลจาก **โหมดทดสอบ (Development/Override)** เป็น **โหมดใช้งานจริง (Production)** เพื่อให้ AI หรือ Developer สามารถดำเนินการได้ทันทีอย่างถูกต้องและปลอดภัย

---

## 📋 ขั้นตอนการเปลี่ยนเป็นระบบเมลจริง (Production)

เมื่อได้รับคำสั่งให้เปลี่ยนเป็นเมลจริง ให้ดำเนินการดังนี้:

### 1. แก้ไขไฟล์ Config ฝั่ง Server
เปิดไฟล์ `.env.development` (หรือ `.env` ที่ใช้งานอยู่บน server) ในโฟลเดอร์ `server/`:

*   **หาบรรทัด:** `TEST_EMAIL_OVERRIDE=wayroad@shindengen.co.th`
*   **สิ่งที่ต้องทำ:** ให้ทำการ **Comment out (#)** หรือ **ลบ** บรรทัดนี้ออก
*   **ตรวจสอบ:** ตรวจสอบให้มั่นใจว่าไม่มีอีเมลทดสอบอื่นค้างอยู่ในตัวแปรนี้

### 2. ตั้งค่า NODE_ENV
*   ตรวจสอบค่า `NODE_ENV` ให้เป็น `production` เพื่อให้ระบบใช้ Logic การดึงผู้รับจริงจาก Database

### 3. Restart Service
หลังจากบันทึกไฟล์ `.env` แล้ว ต้องทำการ Restart NestJS Server เพื่อให้ค่า Config ใหม่ถูกโหลดเข้าไปในระบบ

---

## 🎯 พฤติกรรมของระบบหลังการเปลี่ยน (Expectation)

เมื่อระบบอยู่ในโหมด Production อีเมลจะถูกส่งตามกฎต่อไปนี้:

| ประเภทเหตุการณ์ (Notify Type) | ผู้รับอีเมล (Recipients) |
|-------------------------|------------------------|
| **การขออนุมัติ (Approval)** | ทุกคนในกลุ่ม `GROUP_LEAD` (อ้างอิงจากตาราง `approval_roles`) |
| **การตีกลับให้แก้ไข (Rework)** | ผู้สร้างเอกสาร (อ้างอิงจากฟิลด์ `created_by` ใน header) |
| **การอนุมัติเสร็จสิ้น (Completed)** | ผู้สร้างเอกสาร (อ้างอิงจากฟิลด์ `created_by` ใน header) |

---

## 🔍 วิธีตรวจสอบความถูกต้อง (Verification)

1.  **ตรวจสอบ Log:** ดูในตาราง `email_logs` ฟิลด์ `is_test_override` ต้องเป็น `0` (False)
2.  **ตรวจสอบผู้รับ:** ฟิลด์ `recipient_emails` ต้องแสดงเป็นอีเมลจริงของพนักงาน (เช่น `somchai.p@shindengen.co.th`)
3.  **สถานะการส่ง:** ฟิลด์ `sent_status` ต้องเป็น `SUCCESS`

---

## ⚠️ ข้อควรระวัง
*   ห้ามลบ `EMAIL_SERVICE_URL` เพราะเป็นช่องทางหลักในการเชื่อมต่อกับ SMTP API
*   หากเปลี่ยนแล้วเมลไม่ถึงผู้รับ ให้ตรวจสอบใน `view_employee_all` ว่าพนักงานท่านนั้นมีข้อมูลในคอลัมน์ `email` หรือไม่

---
**สถานะปัจจุบัน:** ยังคงเป็นโหมดทดสอบ (Redirect to Test Email) จนกว่าจะได้รับคำสั่งเปลี่ยนแปลง
